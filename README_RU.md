# Язык программирования Lazy v1.1

**Первый в мире язык программирования с НУЛЁМ ключевых слов!**

## Что нового в v1.1

- **Умная система ввода** — кастомные подсказки, несколько переменных, итеративный ввод с `+?`
- **Цепочки if-else-if** — полноценная условная логика с `??`
- **Быстрые функции** — однострочные определения функций с `~>`
- **Расширенные операции со списками** — реверс, сортировка, поиск, уникальные элементы
- **Полная поддержка Unicode** — переменные на ЛЮБОМ языке (русский, 中文, العربية и т.д.)
- **Стрелочные параметры** — использование `->` для читаемости
- **Inline-ввод** — `+??` для временного ввода прямо в выражениях
- **Операторы мутации** — суффикс `*` для изменения списков на месте
- **Блочная область видимости** — `{}` для чёткой структуры кода

## Что такое Lazy?

Lazy — это экспериментальный язык программирования, который использует **только символы**.  
Никаких ключевых слов. Вообще.

Если ты понимаешь стрелки и символы — ты можешь писать на Lazy.

## Установка и настройка

### Сборка интерпретатора

1. **Создай проект:**
```bash
cargo new lazy
cd lazy
```

2. **Замени `src/main.rs`** кодом интерпретатора Lazy

3. **Собери исполняемый файл:**
```bash
cargo build --release
```

4. **Исполняемый файл будет находиться здесь:**
- Windows: `target\release\lazy.exe`
- Linux / Mac: `target/release/lazy`

### Запуск программ Lazy

**Вариант 1: запуск `.lazy` файла**
```bash
# Windows
.\target\release\lazy.exe program.lazy

# Linux / Mac
./target/release/lazy program.lazy
```

**Вариант 2: интерактивный REPL**
```bash
# Windows
.\target\release\lazy.exe

# Linux / Mac
./target/release/lazy
```

Вводи код, затем напиши `run` для выполнения.

## Символьная система

Lazy не содержит **ни одного ключевого слова**. Всё реализовано через символы:

| Символ | Назначение | Подсказка            |
|------|-----------|----------------------|
| `+?` | Ввод с подсказкой | вопрос — значит ввод |
| `+??` | Inline-ввод | временный ввод       |
| `?` | if / else-if / else | условие              |
| `@` | Цикл | повторение           |
| `->` | return / параметр | стрелка              |
| `=>` | Начало функции | жирная стрелка       |
| `~>` | Быстрая функция | шорткат              |
| `}` | Конец блока | закрытие             |
| `>>` | for-each | проход               |
| `=` | Присваивание | равно                |
| `*` | Мутация | изменить оригинал    |

## Встроенные символьные функции

Вместо именованных функций вроде `len()` Lazy использует **символьные операторы**.

### Базовые операции
| Символ | Назначение | Пример |
|------|-----------|--------|
| `#(x)` | Длина | `#(mylist)` → 5 |
| `$(x)` | В строку | `$(42)` |
| `~(x)` | В число | `~("42")` |
| `!(bool)` | Логическое НЕ | `!(yes)` |
| `?=(max)` | Случайное число | `?=(10)` |

### Операции со списками
| Символ | Назначение        |
|------|-------------------|
| `^(list -> val)` | Добавить элемент  |
| `v(list)` | Удалить последний |
| `<>(list)` | Перевернуть       |
| `++(list)` | Сортировка ↑      |
| `--(list)` | Сортировка ↓      |
| `><(list -> val)` | Проверка наличия  |
| `<<(list)` | Удалить дубликаты |

## Операторы мутации

По умолчанию операции **не изменяют** исходные данные.

```lazy
numbers = [1 2 3]
new_numbers = ^(numbers -> 4)
numbers
```

Используй `*`, чтобы изменить оригинал:

```lazy
^(numbers -> 4)*
numbers
```

Работает со всеми операциями списков.

## Синтаксис

### Вывод

```lazy
"Hello World"
42
```

Никакого `print`.

### Переменные (Unicode)

```lazy
x = 5
имя = "Марат"
名前 = "太郎"
```

### Арифметика

```lazy
a = 10 + 5
b = 20 - 3
```

### Ввод

```lazy
+? name : "Как тебя зовут? "
```

### Функции

```lazy
add(a b) ~> a + b
```

### Условия

```lazy
? x > 5 {
  "x большой"
}
```

### Циклы

```lazy
@ counter < 5 {
  counter = counter + 1
}
```

## Философия

**Почему без ключевых слов?**

Ключевые слова привязывают язык к английскому. Символы:
- универсальны
- визуальны
- минималистичны
- интуитивны
- дружелюбны к Unicode

Lazy создан для:
- обучения
- экспериментов
- прототипирования
- мультиязычных разработчиков

## Ограничения (осознанные)

- Нет классов
- Нет импортов
- Нет файлового ввода/вывода
- Только динамическая типизация

Это не баги. Это философия.

Для более подробной информации о синтаксисе языка перейди на английскую версию
